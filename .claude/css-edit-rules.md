
CSS編集ルール
==================================================

このリポジトリ固有のルール。命名規則は含まない。


CSS記述ルール
--------------------------------------------------

* 編集対象は `src/`
	`dist/` は生成物（コピー＋コンパイル）。コミット済みだが直接編集しない
* 全ての `.scss` の先頭に表題ヘッダーを付ける
	* setext h1 / `=` は50個 / 1行目は空 `//`
	* `===` 直下にファイルの説明を書く（任意・複数行可）。ヘッダー末尾は空 `//` で締める
	* 記述例：  
		```scss
		//
		// _btn.scss
		// ==================================================
		// ボタンのスタイル（説明は任意）
		//
		```
* **インデントはタブ**で統一
* `@layer` 宣言とGlibalの取り込みは `src/styles/global.scss` のみ
	* `meta.load-css` を利用し、`@layer <layer-name>{}` でラップ
	* 各サブディレクトリの `_index.scss` は `@forward` で個別ファイルを繋ぐ
	* スタイルは個別ファイルに書く
* `load-path` は `node_modules` と `src/styles`
	* 内部参照は `@use "abstracts/tools"`
	* 外部CSS は `@use "kiso.css/kiso.css"`
* `@font-face` の `url()` は出力先 `dist/assets/css/` からの相対
  * ＝ `../fonts/...`（Sass は `url()` を書き換えないため）
